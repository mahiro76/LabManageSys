# 实验室考勤管理系统 - 架构与代码分析

## 一、数据库设计：主键、外键与表关系

系统使用 MySQL 数据库 `lab_attendance_system`，共 4 张表。

### 表结构概览

```
┌───────────────────┐       ┌──────────────────────┐
│   departments     │       │      members         │
│───────────────────│       │──────────────────────│
│ PK  department_id │──┐    │ PK  member_id(VARCHAR)│
│     department_name│  │   │     name             │
│     description   │  │    │     gender,age       │
└───────────────────┘  │    │     department_id ────┤ FK
                       │    └──────────────────────┘
                       │              │
                       │              │ FK
                       │    ┌─────────┴────────────┐
                       │    │  attendance_records  │
                       │    │──────────────────────│
                       │    │ PK  attendance_id     │
                       │    │     member_id ────────┤ FK
                       │    │     attendance_date   │
                       │    │     status            │
                       │    │     check_in/out_time │
                       │    │ UNIQUE(member_id,date)│
                       │    └──────────────────────┘
                       │              │
                       │              │ FK
                       │    ┌─────────┴────────────┐
                       │    │    user_accounts      │
                       │    │──────────────────────│
                       │    │ PK  user_id           │
                       │    │     username (UNIQUE) │
                       │    │     password_hash     │
                       │    │     role              │
                       │    │     member_id ────────┤ FK
                       │    │     is_active         │
                       └────┘                      │
                            └──────────────────────┘
```

### 主键 (Primary Key)

| 表名 | 主键字段 | 类型 |
|------|----------|------|
| departments | `department_id` | INT AUTO_INCREMENT |
| members | `member_id` | VARCHAR(32) 手动输入 |
| attendance_records | `attendance_id` | INT AUTO_INCREMENT |
| user_accounts | `user_id` | INT AUTO_INCREMENT |

### 外键 (Foreign Key)

| 外键名 | 来源表 | 来源字段 | 目标表 | 目标字段 | ON DELETE | ON UPDATE |
|--------|--------|----------|--------|----------|-----------|-----------|
| FK_Members_Department | members | department_id | departments | department_id | SET NULL | CASCADE |
| FK_Attendance_Member | attendance_records | member_id | members | member_id | **CASCADE** | CASCADE |
| FK_Users_Member | user_accounts | member_id | members | member_id | **SET NULL** | CASCADE |

**关键外键行为说明：**
- `members.department_id → departments.department_id`：删除部门时，该部门下的成员的 `department_id` 被设为 NULL（SET NULL），成员记录保留
- `attendance_records.member_id → members.member_id`：删除成员时，该成员的所有考勤记录一并删除（CASCADE）
- `user_accounts.member_id → members.member_id`：删除成员时，关联账号的 `member_id` 设为 NULL（SET NULL），账号保留

### 唯一约束

- `departments.department_name`：UNIQUE，部门名不能重复
- `members.member_id`：PRIMARY KEY，成员编号唯一
- `attendance_records`：UNIQUE KEY `UkMemberDate` (member_id, attendance_date)，**同一成员同一天只能有一条考勤记录**
- `user_accounts.username`：UNIQUE，用户名唯一
- `user_accounts.member_id`：UNIQUE，一个成员只能关联一个账号

### 查看数据库代码的位置

- **表结构创建**：`db.py` 第 101-172 行，`_CreateTables()` 方法
- **完整 SQL 查询**：`db.py` 整个文件，`MySqlManager` 类的所有方法
- **数据模型定义**：`models.py` 文件，定义 `DatabaseConfig` 和 `LoginUser` 数据类

## 三、关于删除数据的问题

### Q: 在系统删数据，数据库里为什么不会删？

**答案：系统中删的是"账号"，不是"成员"。** 有两条删除路径：

#### 路径 A：用户管理 → 删除用户

调用的代码链：
```
UserPage._OnDelete() → self._Db.DeleteUser(user_id)
```
对应的 SQL（`db.py` 第 455-457 行）：
```sql
DELETE FROM user_accounts WHERE user_id = %s
```
这只删除 `user_accounts` 表中的登录账号，**不会删除 `members` 表中的成员档案**。

#### 路径 B：成员管理 → 删除成员

调用的 SQL（`db.py` 第 233-236 行）：
```sql
DELETE FROM members WHERE member_id = %s
```
这会删除成员 + 连带删除该成员的考勤记录（外键 ON DELETE CASCADE），但关联的用户账号只是 `member_id` 被设为 NULL，账号还在。

#### 结论
- 删除"用户"（账号）≠ 删除"成员"（档案）
- 删除"成员"才会删数据库中的实际数据
- 建议：如果真的要删除某个人的全部数据，应该在成员管理中删除

## 四、CSV/Excel 导入

### Q: 为什么只能导入 CSV，不能手动批量添加？

**实际上系统支持两种导入方式：**

#### 4.1 导入 Excel (.xlsx)

`MemberPage._OnImport()`（第 ~430-490 行）：使用 `openpyxl` 库读取 `.xlsx` 文件，并逐行插入到 `members` 表。

#### 4.2 导入 CSV

`MemberPage._OnImportCsv()`（第 ~492 行起）：使用 `csv` 模块读取 `.csv` 文件。

#### 为什么不提供"手动批量添加"？
因为**批量录入天然适合用表格文件**（Excel/CSV）：
- 用户可以在 Excel 中一次性编辑多条数据
- 导入时自动校验手机号格式、年龄范围、性别取值
- 支持已知错误时继续导入其他行（部分成功部分失败）

## 五、学生端打卡时间/权限

### 学生考勤页面的行为

学生登录后看到的是**同一个** `AttendancePage` 类（`ui.py` 第 736 行），但行为受角色限制：

1. **只能看自己的记录**：`member_id` 锁定为自己的编号，文本框只读
2. **没有新增/修改/删除按钮**：因为按钮只对 `ROLE_ADMIN` 可见
3. **看的是"我的考勤"标签**：Tab 标题从"考勤管理"变为"我的考勤"

### 打卡时间字段

考勤记录包含：
- `check_in_time`（签到时间）：TIME 类型，可为 NULL
- `check_out_time`（签退时间）：TIME 类型，可为 NULL
- `status`：正常 / 迟到 / 早退 / 请假

学生端**没有自助打卡功能**——签到/签退时间由 Admin 在考勤管理页面录入。

## 七、Hash (哈希) 的作用

你问的"mash"应该是指 **Hash（哈希）**。

### 密码哈希函数

`models.py` 第 49-52 行：

```python
def HashPassword(Password: str) -> str:
    """对密码进行 SHA-256 哈希"""
    import hashlib
    return hashlib.sha256(Password.encode("utf-8")).hexdigest()
```

#### 作用：
1. **密码不存明文**：数据库中存的是 64 位十六进制哈希值，而不是原始密码
2. **登录验证**：用户输入的密码经 SHA-256 哈希后，与数据库中的哈希值比对
3. **安全性**：即使数据库泄露，攻击者也拿不到原始密码

#### 使用位置：
- 创建用户时：`HashPassword(Password)` 存入 `password_hash` 字段
- 登录时：`WHERE password_hash = %s`，传入 `HashPassword(用户输入的密码)`
- 修改密码时：同样用哈希值比较和更新

#### 1. 数据库操作层（`db.py`）

def EnsureConnection(self):

except mysql.connector.Error as e:

## 九、教师端搜索学生：匹配方式

### 成员搜索

`db.py` 第 296-327 行，`GetMembers()` 方法：

```python
if Keyword:
    SqlText += """
        AND (m.member_id LIKE %s OR m.name LIKE %s)
    """
    LikeValue = f"%{Keyword}%"
```

**不是用正则表达式**，而是用 SQL 的 `LIKE` 模糊匹配：
- `LIKE '%keyword%'` → 包含关键字即匹配
- 同时搜索 `member_id`（成员编号）和 `name`（姓名）
- 例：搜索"张" → 匹配"张同学"、"张三"

### 考勤记录搜索

`db.py` 第 331-363 行，`GetAttendanceRecords()` 同样使用 `LIKE` 模糊匹配成员编号。

### 正则表达式（Regex）在项目中的实际使用

正则只用在这两个地方：

1. **手机号校验**（`models.py` 第 23-24 行）：
```python
DEFAULT_PHONE_PATTERN = re.compile(r"^1[3-9]\d{9}$")
```
匹配规则：以 1 开头，第二位 3-9，后面 9 位数字，共 11 位。

2. **提取 MySQL 临时密码**（`mysql_service.py` 第 107 行）：
```python
match = re.search(r"temporary password.*?:\s+(\S+)", content, re.IGNORECASE)
```
从 MySQL 错误日志中提取临时 root 密码。

### 什么是正则表达式？

正则表达式（Regular Expression，简称 regex/re 正则）是一种**文本模式匹配工具**，用特殊符号定义字符串的匹配规则：

| 符号 | 含义 | 示例 |
|------|------|------|
| `^` | 开头 | `^1` 以1开头 |
| `$` | 结尾 | `\d{9}$` 以9位数字结尾 |
| `\d` | 数字 | `\d{9}` 9位数字 |
| `[3-9]` | 3到9的任意数字 | `[3-9]` |
| `.*?` | 任意字符（非贪婪） | 用于提取密码 |
| `\S+` | 非空白字符序列 | 匹配密码值 |
